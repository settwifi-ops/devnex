1. Quick summary (one-line)

Each signal object represents a coin detected by the screener with raw metrics (price, volume, RSI), derived analytics (health_score, trend_strength, momentum_phase), and a smart_confidence score which the enggine uses to rank signals. Use health_score and trend_strength for quality checks; follow risk_level and momentum_phase for tactical actions.

4. How the enggine decides priority / urgency

The enggine computes a combined priority using enhanced_score, smart_confidence, health_score, trend_strength, absolute performance and appearance count. The aggregator returns a category (e.g. üî• HIGHEST, üöÄ HIGH, üìà MEDIUM-HIGH, ...). Use HIGHEST/HIGH to trigger immediate attention and possible allocation; MEDIUM to watch; LOW to ignore.

5. Practical reading ‚Äî decision rules (examples)

  1. Buy candidate (conservative):
    • health_score > 75 AND trend_strength > 60
    • smart_confidence > 65 AND multi_tf_confirmation == "BULL"
    • order_block_alignment == "NEAR_SUPPORT" preferred
  2. Speculative entry (higher risk):
    • enhanced_score > 65 AND volume_spike_ratio > 2.0 AND health_score > 60
    • If momentum_phase == "PARABOLIC" ‚Üí consider taking partial profits or setting tight SL
  3. Avoid / exit:
    • risk_level == "HIGH" OR health_score < 45
    • performance_since_first > 40% with momentum_phase == "DISTRIBUTION" (possible top)

These are examples ‚Äî adapt thresholds to your strategy and position sizing rules.

6. Where values come from in the code (reference)

  • Display payload / DB fields: function get_display_data(signal) composes all display fields.
  • Health score: computed in _calculate_enhanced_health_score ‚Äî weighted factors: performance, score improvement, volume trend, momentum trend, price trend, appearance_count, consistency.
  • Trend strength: computed in _calculate_enhanced_trend_strength, uses performance magnitude, volume strength, momentum strength, RSI delta, appearance strength.
  • Smart confidence: computed in calculate_enhanced_smart_confidence ‚Äî aggregates market_context, volume_profile, momentum regime, order blocks, etc.

7. Quick troubleshooting

  • Missing fields: If a signal lacks a field, check the data pipeline step that extracts klines/volumes ‚Äî default fallbacks exist in the code (e.g. default RSI=50, volume_spike=1).
  • Low confidence even with pump: Market context (BTC trend & Fear&Greed) can reduce smart_confidence. See MarketContextAnalyzer.
  • Order block detection noisy: OrderBlockLiteAnalyzer requires min lookback; if klines < threshold it returns empty. Increase lookback data.
  • Signals not showing in UI: The screener saves UI signals into self.last_ui_signals and DB via the database manager ‚Äî check DB insert logs for errors.

8. Glossary (short)

  • Health score: composite gauge of signal robustness (0‚Äì100).
  • Trend strength: how strong the detected trend is (0‚Äì100).
  • Smart confidence: model confidence combining market, volume and orderblock signals.
  • Order block: historical area of price rejection/absorption (demand/supply).
  • Momentum phase: lifecycle stage of the move (guides trade urgency).
